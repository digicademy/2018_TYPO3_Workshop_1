Das Typo3 CMS

Was ist Typo3?

  • Open Source Content Management System (CMS) der Enterprise-Klasse
  • Erfunden von Kasper Skårhøj 1997
  • Erste öffentliche Version 3.0 2001
  • Kollektive Weiterentwicklung von Typo3 Community
  • Koordination und nachhaltige Entwicklung durch TYPO3 Association
  • Diverse (nicht-)kommerzielle Use Cases
  • Typo3 ist ein Content Management Framework (CMF)

Warum ein CMS/CMF?

  • CMS
    • Trennung von Daten, Struktur, Design
    • Einfache Pflege redaktioneller Inhalte
    • Saubere Organisation von Inhalten
    • Reduzierung von Fehleranfälligkeit der Struktur
  • CMF
    • Macht Stabile CMS-Struktur flexibel
    • Quasi uneingeschränkte Erweiterbarkeit
    • Einheitliche Entwickliungsstandards erhöhen Nachhaltigkeit
    • Framework-Architektur erhöht Schnelligkeit
    • Geringere Fehleranfälligkeit im Team-Workflow

Warum Typo3?

  • Out-of-the-box features
    • Sicherheit
    • Stabilität
    • Long-term support
    • Redaktionsbackend
    • Multiple Accounts
    • Mehrsprachigkeit
    • …
  • Flexibilität
    • Individuelle Frontends
    • PHP Core
    • Diverse Datenbankanbindungen
    • Upgradezyklus
    • …

Das Typo3 Backend

Seitenbaum und Module

Page Module

Zeigt eine strukturierte Ansicht, der einer Seite zugeordeten Content Elemente und Plugins. Die Struktur wird in sog. Backend Layouts festgelegt.

List Module

Zeigt alle Datensätze, die dieser Seite zugeordnet sind (Seiten, Content Elemente, Spezifische Datensätze wie z.B. News, Plugins etc.).

Das Typo3 Backend

Extensions

Extensions ermöglichen es den Typo3 Core individuell zu erweitern. Gleichzeitig können aufgrund der API Architektur Core-Funktionalitäten von Typo3 nicht kompromitiert werden. Es gibt spezielle System Extensions, die Typo3 out-of-the-box enthält. Im TER stehen weitere Extensions von Entwickler*innen aus aller Welt bereit und können dort auch einfach geteilt werden.
